# Konflux Access Setup

Run these steps inside a **manual Ambient Code session** (open the `koku` workspace in the Ambient Code UI and start a session). All commands below execute in the session's terminal — not on your local machine.

> **Before you start:** Make sure `KONFLUX_TOKEN` is already set in **Workspace Settings → Custom Environment Variables**. The steps below use it directly from the environment — no manual file paste needed.

---

**Step 1: Create artifacts directory**

```bash
mkdir -p /workspace/artifacts
```

**Step 2: Download kubectl if not already present**

```bash
if [ ! -f /workspace/artifacts/kubectl ]; then
    curl -Lo /workspace/artifacts/kubectl https://dl.k8s.io/release/v1.28.0/bin/linux/amd64/kubectl
    chmod +x /workspace/artifacts/kubectl
fi
/workspace/artifacts/kubectl version --client
```

**Step 3: Create the kubeconfig for Konflux**

```bash
/workspace/artifacts/kubectl config --kubeconfig=/workspace/artifacts/kubeconfig-konflux.yaml \
    set-cluster konflux \
    --server=https://api.stone-prd-rh01.pg1f.p1.openshiftapps.com:6443 \
    --insecure-skip-tls-verify=true

/workspace/artifacts/kubectl config --kubeconfig=/workspace/artifacts/kubeconfig-konflux.yaml \
    set-credentials konflux-bot-0 \
    --token=$KONFLUX_TOKEN

/workspace/artifacts/kubectl config --kubeconfig=/workspace/artifacts/kubeconfig-konflux.yaml \
    set-context default \
    --cluster=konflux \
    --user=konflux-bot-0 \
    --namespace=cost-mgmt-dev-tenant

/workspace/artifacts/kubectl config --kubeconfig=/workspace/artifacts/kubeconfig-konflux.yaml \
    use-context default
```

**Step 4: Download kubectl-ka (KubeArchive CLI) for accessing archived pipeline logs**

```bash
if [ ! -f /workspace/artifacts/kubectl-ka ]; then
    curl -Lo /workspace/artifacts/kubectl-ka \
        https://github.com/kubearchive/kubearchive/releases/latest/download/kubectl-ka-linux-amd64
    chmod +x /workspace/artifacts/kubectl-ka
fi
/workspace/artifacts/kubectl-ka version
```

**Step 5: Verify all three KubeArchive endpoints**

```bash
export KUBECONFIG=/workspace/artifacts/kubeconfig-konflux.yaml
TOKEN=$KONFLUX_TOKEN
KA_HOST="https://kubearchive-api-server-product-kubearchive.apps.stone-prd-rh01.pg1f.p1.openshiftapps.com"
NS="cost-mgmt-dev-tenant"

# PipelineRuns (should return items)
curl -sk -H "Authorization: Bearer $TOKEN" "$KA_HOST/apis/tekton.dev/v1/namespaces/$NS/pipelineruns?limit=1" \
  | python3 -c "import json,sys; d=json.load(sys.stdin); print('PipelineRuns:', len(d.get('items',[])))"

# TaskRuns (should return items — requires ci-triager-taskrun-reader role)
curl -sk -H "Authorization: Bearer $TOKEN" "$KA_HOST/apis/tekton.dev/v1/namespaces/$NS/taskruns?limit=1" \
  | python3 -c "import json,sys; d=json.load(sys.stdin); print('TaskRuns:', len(d.get('items',[])), '| err:', d.get('message','none'))"

# Pods (should return items)
curl -sk -H "Authorization: Bearer $TOKEN" "$KA_HOST/api/v1/namespaces/$NS/pods?limit=1" \
  | python3 -c "import json,sys; d=json.load(sys.stdin); print('Pods:', len(d.get('items',[])), '| err:', d.get('message','none'))"
```

All three should return items count > 0. If TaskRuns/Pods return `msg: unauthorized`, the
`ci-triager-taskrun-reader` RoleBinding is missing — see RBAC section below.

## RBAC Requirements

The `konflux-bot-0` ServiceAccount needs two RoleBindings in the `cost-mgmt-dev-tenant` namespace:

| RoleBinding | Role | Grants |
|---|---|---|
| `ci-triager-viewer-binding` | `appstudio-contributor-user-actions` | PipelineRuns, Snapshots, etc. |
| `ci-triager-taskrun-reader-binding` | `ci-triager-taskrun-reader` | TaskRuns, Pods (for IQE logs) |

These manifests live in the `konflux-release-data` GitOps repository under `tenants-config/cluster/stone-prd-rh01/tenants/cost-mgmt-dev-tenant/ci-triager/`. **Do not apply them with `oc apply` directly** — add or update the YAML files in that repo and ArgoCD will deploy them automatically.

The `ci-triager-taskrun-reader` Role definition:

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: ci-triager-taskrun-reader
  namespace: cost-mgmt-dev-tenant
rules:
- apiGroups: ["tekton.dev"]
  resources: ["taskruns"]
  verbs: ["get", "list", "watch"]
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list"]
```

## Token Notes

- The token generated by `oc create token ... --duration=8760h` uses the cluster's OIDC issuer.
  This is the **correct token type** for KubeArchive on this cluster.
- Do **not** use `--audience=https://kubernetes.default.svc` — that audience is rejected by the
  cluster's SubjectAccessReview and causes KubeArchive to return 0 items.
- The token is stored in **Ambient Code → Workspace Settings → Custom Environment Variables → `KONFLUX_TOKEN`**. It is injected automatically into every session — no manual file step needed.
- The token expires after ~1 year; regenerate with `oc create token konflux-bot-0 -n cost-mgmt-dev-tenant --duration=8760h` and update the value in Workspace Settings.
